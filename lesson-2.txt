1: Giải thích câu lệnh
a. git init
   - Khởi tạo một repository Git mới trong thư mục hiện tại
   - Tạo thư mục ẩn .git chứa tất cả metadata và lịch sử của repository
   - Sau khi chạy lệnh này, thư mục sẽ trở thành một Git repository
b. git add <file_name>
   - Thêm một file cụ thể vào staging area (vùng chờ)
   - File được đánh dấu để chuẩn bị cho commit tiếp theo
   - Ví dụ: git add file1.txt sẽ thêm file1.txt vào staging area
c. git add .
   - Thêm TẤT CẢ các file đã thay đổi hoặc file mới trong thư mục hiện tại vào staging area
   - Dấu chấm (.) đại diện cho thư mục hiện tại
   - Tiện lợi khi muốn add nhiều file cùng lúc
d. git commit -m "message"
   - Tạo một commit (lần lưu) mới với tất cả các file trong staging area
   - -m cho phép thêm commit message ngay trong câu lệnh
   - Commit message nên mô tả ngắn gọn những thay đổi đã thực hiện
